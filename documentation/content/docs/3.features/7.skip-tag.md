# Skipping Tests

TestFlowKit allows you to temporarily exclude scenarios or entire features from execution by applying the `@skip` tag. Skipped scenarios are excluded from all test runs — including `@BeforeAll`, main, and `@AfterAll` phases.

## How It Works

Any scenario or feature tagged with `@skip` is filtered out during parsing, before execution begins. A warning is logged for each skipped scenario so you always have visibility into what was excluded.

## Skipping a Single Scenario

Apply `@skip` directly on a scenario to exclude only that scenario:

```gherkin
Feature: User account

  Scenario: Successful login
    Given I prepare a request for the "login" endpoint
    When I send the request
    Then the response status code should be 200

  @skip
  Scenario: Forgot password
    Given I prepare a request for the "forgot_password" endpoint
    When I send the request
    Then the response status code should be 200
```

In this example, `Forgot password` is excluded and `Successful login` runs normally.

## Skipping an Entire Feature

Apply `@skip` at the feature level to exclude all scenarios in that file:

```gherkin
@skip
Feature: Experimental payments

  Scenario: Pay with crypto
    Given I prepare a request for the "crypto_payment" endpoint
    When I send the request
    Then the response status code should be 200

  Scenario: Pay with voucher
    Given I prepare a request for the "voucher_payment" endpoint
    When I send the request
    Then the response status code should be 200
```

Both scenarios are excluded because the `@skip` tag is inherited by all scenarios in the feature.

## Warning Logs

When scenarios are skipped, TestFlowKit emits a warning for each one:

```
⚠  Skipping scenario: "Forgot password" in feature: "User account"
```

This makes it easy to spot accidentally skipped tests during CI runs.

## Use Cases

- ✅ Temporarily disable a flaky test without deleting it
- ✅ Exclude work-in-progress scenarios from CI
- ✅ Skip a whole feature while its backing API is under development
