# Frontend Testing

TestFlowKit provides powerful browser automation capabilities for testing web applications. This guide covers all frontend testing features.

## Browser Drivers

TestFlowKit supports two browser automation drivers:

### Rod Driver

The default driver using [go-rod](https://go-rod.github.io/), providing fast and reliable Chrome/Chromium automation.

**When to use:**
- Default choice for most testing scenarios
- Fast execution with minimal overhead
- Stable and well-tested
- Lower resource consumption

### Playwright Driver

Alternative driver using [playwright-go](https://github.com/playwright-community/playwright-go), offering advanced browser automation features.

**When to use:**
- Need advanced browser features
- Cross-browser testing requirements
- Complex web application scenarios
- Modern web platform capabilities

**Prerequisites:**

To use the Playwright driver, run the install command:

```bash
tkit install --config=config.yml
```

This automatically installs Chromium and system dependencies.

::alert{type="warning"}
**Requirements:** The Playwright driver requires [Go 1.19+](https://golang.org/dl/) and an internet connection for installation.
::

### Configuration

Specify the driver in your `config.yml`:

```yaml
frontend:
  driver: "rod"  # or "playwright"
  default_timeout: 10000
  headless: false
  screenshot_on_failure: true
```

::alert{type="warning"}
The `driver` field is **required** when the frontend block is defined in your configuration.
::

## Best Practices

### 1. Use Descriptive Element Names

```yaml
# Good
elements:
  submit_registration_form: "#register-submit"
  
# Bad
elements:
  btn1: "#register-submit"
```

### 2. Wait Appropriately

```gherkin
# Good: Wait for specific element
When the user clicks the "load_more" button
And the user waits for the "new_items" to be visible

# Bad: Arbitrary sleep
When the user clicks the "load_more" button
And the user waits for 5 seconds
```

### 3. Verify State After Actions

```gherkin
# Always verify the action had the expected effect
When the user clicks the "submit" button
Then the "success_message" should be visible
And the current URL should contain "/confirmation"
```
