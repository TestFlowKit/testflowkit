# Step Definitions

TestFlowKit steps are fixed Gherkin sentences registered in the framework. You cannot add custom steps without modifying the Go codebase.

For the full searchable catalog with parameters and examples, use the **[Step Definitions Browser](/sentences)**.

## Categories

| Category | Doc | Examples |
|----------|-----|----------|
| **Navigation** | [Frontend Testing](/docs/features/frontend-testing) | `the user goes to the "page" page` |
| **Forms & clicks** | [Frontend Testing](/docs/features/frontend-testing) | `the user enters "..." into the "field" field` |
| **Assertions** | [Frontend Testing](/docs/features/frontend-testing) | `the "element" should be visible` |
| **REST API** | [API Testing](/docs/features/api-testing) | `I prepare a request to "api.endpoint"` |
| **GraphQL** | [API Testing](/docs/features/api-testing) | `I set the following GraphQL variables:` |
| **Variables** | [Variables](/docs/features/variables) | `I store the value "..." into "name" variable` |
| **Random data** | [Random Data](/docs/features/random-data-generation) | `{{ rand:uuid }}` inline in any step |
| **Macros** | [Macros](/docs/features/macros) | `${variable}` in `@macro` scenarios |
| **Global hooks** | [Global Hooks](/docs/features/global-hooks) | `@BeforeAll`, `@AfterAll` features |

## Parameter types

| Type | Syntax |
|------|--------|
| String | `"value"` in quotes |
| Table | Two-column rows below the step |
| Doc string | `"""` block below the step (JSON bodies, long text) |

```gherkin
When the user enters "john@example.com" into the "email" field

And I set the following path parameters:
  | id | 123 |

And I set the request body to:
  """
  { "name": "John" }
  """
```
